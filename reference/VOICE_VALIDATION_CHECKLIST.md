# VOICE MARKER VALIDATION CHECKLIST
## I→D→G Quality Control Tool

**Date:** January 19, 2026  
**Purpose:** Systematic validation of generated content against I→D→G voice architecture  
**For:** Opus 4.5 validation, Sonnet 4.5 self-check, Joshua review

---

## HOW TO USE THIS CHECKLIST

1. **After generating content:** Run through all checkboxes
2. **Any ❌:** Revise before submitting
3. **Ambiguous cases:** Flag for human review
4. **Final validation:** All checkboxes must be ✅

---

## SECTION 1: PHASE STRUCTURE CHECK

### Phase Balance
- [ ] Phase 1 (Invention Opening): 10-15% of content
- [ ] Phase 2 (Discernment Middle): 60-70% of content
- [ ] Phase 3 (Galvanizing Close): 15-25% of content

### Phase Order
- [ ] Content flows I → D → G (not scrambled)
- [ ] Transitions between phases are clear
- [ ] Each phase accomplishes its specific function

---

## SECTION 2: PHASE 1 (INVENTION) VALIDATION

### Content Requirements
- [ ] Opens with concrete story or scene (NOT abstract definition)
- [ ] Curiosity gap created (reader wants to know more)
- [ ] Novel framing previewed (what reader will discover)
- [ ] Stakes hinted at (why this matters)

### Voice Markers PRESENT (at least 2):
- [ ] "Here's what's actually happening..."
- [ ] "Consider a different map..."
- [ ] "There's a pattern here..."
- [ ] "What if we looked at it this way..."
- [ ] "The synthesis reveals..."

### Voice Markers ABSENT (none of these):
- [ ] NO "In this chapter, we will..."
- [ ] NO "One might argue..."
- [ ] NO "Studies suggest..."
- [ ] NO "The reader should observe..."

### Reader Response Test
- [ ] Reader will think: "I've never seen it this way before"

---

## SECTION 3: PHASE 2 (DISCERNMENT) VALIDATION

### Content Requirements
- [ ] Framework tested through concrete examples
- [ ] Reader invited to verify in their own experience
- [ ] Multiple examples across domains (triangulation)
- [ ] Limits and edge cases acknowledged
- [ ] Diagnostic questions provided

### Voice Markers PRESENT (at least 3):
- [ ] "Think of a time when..."
- [ ] "You've probably experienced..."
- [ ] "Notice how this appears in..."
- [ ] "Test this against your own..."
- [ ] "Where have you seen..."
- [ ] "The framework holds here, but..."

### Voice Markers ABSENT (none of these):
- [ ] NO "This proves that..."
- [ ] NO "You must therefore conclude..."
- [ ] NO "Obviously..."
- [ ] NO "As I've demonstrated..."

### Verification Structure
- [ ] Examples precede claims (not just illustrations after)
- [ ] Reader can test framework themselves
- [ ] Questions are genuine invitations, not rhetorical

### Reader Response Test
- [ ] Reader will think: "Yes, I recognize this pattern"

---

## SECTION 4: PHASE 3 (GALVANIZING) VALIDATION

### Content Requirements
- [ ] Implications stated clearly ("This means...")
- [ ] Rally-ready summary provided (shareable in 30 seconds)
- [ ] Stakes made explicit (why this matters)
- [ ] Reader equipped to share ("The next time...")
- [ ] Connection to next chapter (if applicable)

### Voice Markers PRESENT (at least 2):
- [ ] "This means..."
- [ ] "The implication is..."
- [ ] "You can recognize this when..."
- [ ] "The next time someone..."
- [ ] "This equips you to..."
- [ ] "The pattern reveals..."

### Voice Markers ABSENT (none of these):
- [ ] NO "You should do..."
- [ ] NO "Apply this by..."
- [ ] NO "Follow these steps..."
- [ ] NO "Your action plan..."
- [ ] NO "Complete this when..."
- [ ] NO "Implementation checklist..."

### Reader Response Test
- [ ] Reader will think: "I know who I want to tell about this"

---

## SECTION 5: ABSOLUTE PROHIBITIONS CHECK

### Prescriptive Language (MUST BE ZERO)

Scan content for these phrases. **Any occurrence = FAIL:**

- [ ] NO instances of "You should..."
- [ ] NO instances of "You must..."
- [ ] NO instances of "Apply this framework by..."
- [ ] NO instances of "Follow these steps..."
- [ ] NO instances of "Complete this stage when..."
- [ ] NO instances of "Here's your action plan..."
- [ ] NO instances of "Implementation checklist..."
- [ ] NO instances of "Do this on Monday..."
- [ ] NO instances of "Your homework is..."

### Academic Hedging (MUST BE MINIMAL)

- [ ] NO excessive "One might argue..."
- [ ] NO "It's interesting to note..."
- [ ] NO "Studies suggest..." (without concrete application)
- [ ] NO "The discerning reader will observe..."

### Passive Voice (SHOULD BE RARE)

- [ ] Minimal "It can be seen that..."
- [ ] Minimal "It should be noted..."
- [ ] Active voice dominates

---

## SECTION 6: CONCRETE BEFORE ABSTRACT CHECK

### Story-First Structure
- [ ] Chapter opens with concrete story/scene
- [ ] Pattern extracted from story (not imposed on it)
- [ ] Framework named after example shown
- [ ] Additional examples verify pattern
- [ ] Rally-ready summary closes

### FAIL if this order:
- ❌ Framework → Definition → Examples

### PASS if this order:
- ✅ Story → Pattern → Name → Verification → Summary

---

## SECTION 7: RALLY-READY NAMES CHECK

### Required Names Used Correctly
- [ ] "The Ring" (not "circular model" or "framework")
- [ ] "Four Habitats" (not "four categories" or "systems")
- [ ] "Scale Error" (not "misapplication" or "mistake")
- [ ] "Little Commune" (for family/communism)
- [ ] "Commons of Safety" (for safety/socialism)
- [ ] "Surplus Engine" (for market/capitalism)
- [ ] "Emergent Order" (for friendship/anarchy)
- [ ] "Obligation to Excess" (when applicable)

### Name Test
- [ ] Every major concept has a memorable name
- [ ] Names are used consistently throughout
- [ ] Names are explainable in one breath

---

## SECTION 8: READER MILESTONE CHECK

### Milestone Framing
- [ ] Milestone is about RECOGNITION (not action)
- [ ] "Reader can RECOGNIZE X" or "Reader can SEE X"
- [ ] NOT "Reader can APPLY X" or "Reader can DO X"

### Appropriate Milestones:
- ✅ "I can recognize scale errors"
- ✅ "I can see why left/right fails"
- ✅ "I can recognize this habitat"

### Inappropriate Milestones:
- ❌ "I can apply the Ring framework"
- ❌ "I can implement this system"
- ❌ "I can follow the steps"

---

## SECTION 9: DIAGNOSTIC QUESTIONS CHECK

### Questions Present in Phase 2
- [ ] At least 2 verification questions included
- [ ] Questions invite reader self-testing
- [ ] Questions are answerable from reader's experience

### Question Types (at least one of each):
- [ ] Pattern recognition: "Think of a time when..."
- [ ] Scale identification: "Who's involved? What's at stake?"
- [ ] Experience verification: "Where have you seen this?"

### Question Quality
- [ ] Questions are genuine invitations, not rhetorical
- [ ] Reader can actually test the framework
- [ ] Questions don't assume the answer

---

## SECTION 10: FINAL QUALITY GATES

### Gate 1: Voice Authenticity
- [ ] Content sounds like diagnostic guide, not prescription manual
- [ ] Author offers lenses, not instructions
- [ ] Reader is trusted to apply in their own context

### Gate 2: Reader Journey
- [ ] I Phase: Reader curious ("I've never seen it this way")
- [ ] D Phase: Reader verifies ("I recognize this")
- [ ] G Phase: Reader equipped ("I can share this")

### Gate 3: Shareability
- [ ] G reader could explain this to a friend
- [ ] Key insight fits in 30 seconds
- [ ] Rally-ready language is present

### Gate 4: Non-Prescriptive
- [ ] Zero "you should" statements
- [ ] Zero implementation steps
- [ ] Zero action checklists
- [ ] Diagnostic only

---

## SCORING

**Count checkboxes:**
- Sections 1-4: Core requirements
- Sections 5-6: Prohibitions (any failure = revise)
- Sections 7-10: Quality markers

**Thresholds:**
- **PASS:** All Section 5 clear + 90% of other checkboxes
- **REVISE:** Any Section 5 failure OR <80% of other checkboxes
- **MAJOR REVISION:** Multiple Section 5 failures OR <70% overall

---

## QUICK REFERENCE: RED FLAGS

**Instant revision triggers:**
1. Any "You should do X" statement
2. Any implementation checklist
3. Any action steps numbered 1, 2, 3...
4. Opening with abstract definition (not story)
5. No verification questions in Phase 2
6. Prescriptive close instead of recognition close
7. Reader milestone about "applying" instead of "recognizing"
